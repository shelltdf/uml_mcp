# md-mv-web 物理规格

## 技术栈

- Vue 3 + Vite 6 + TypeScript
- 依赖 `@mvwb/core`（`file:../../packages/core`）

## 行为

- **多文档切换**：已打开文件在 **中间编辑列**（`div.editor-column` 内）顶部的 **`header.doc-tabs`** 以 **标签页** 横向排列（短名为路径末段，主按钮 `title` 为完整相对路径）；与 **左右 Dock** 同一行布局中，标签栏仅占 **Dock 之间的主编辑区** 宽度。点击标签切换文档；每个标签右侧有 **关闭（×）**，从内存工作区移除该文件，若关闭当前文档则选中右侧相邻，否则选中左侧。
- **顶栏（Win 风格壳层）**：`header.win-chrome` — **标题条**（应用名 + 版本）、**菜单栏**（文件 / 视图 / 帮助，下拉项与「无全局快捷键」类提示以 `title` 为准）、**工具栏**（左侧：**新建 / 打开 / 打开文件夹 / 保存 / 另存为 / 关闭**；Electron 另显 **磁盘工作区**；**右侧**：**全屏 / 退出全屏**，对根布局 `div.layout` 调用 Fullscreen API，监听 `fullscreenchange` 同步按钮文案）；不列出文件列表。**文件**菜单与工具栏对齐上述命令；**保存** `Ctrl+S`、**另存为** `Ctrl+Shift+S`、**关闭当前** `Ctrl+W`（焦点不在可编辑区时）；关闭标签前有**未保存**确认（与内存 `savedBaseline` 比对）。
- **底栏**：`footer.statusbar` — 左侧摘要（最近一条程序日志或就绪提示），右侧壳类型、已打开文档数、当前路径；**点击状态栏**打开 **日志窗口**（纯文本历史、`复制全文`）。
- **`blockOnly` 模式**（Electron 块编辑 URL）：隐藏顶栏与底栏，主区全宽。
- **mv-model-sql 语义**（与 `@mvwb/core` 一致）：每个 `` ```mv-model-sql `` 块为 **一个 Model 组**（`tables[]` 多张子表）；围栏级 `id` 文件内唯一。解析与列/行校验见 `md-mv-core`。**mv-model-sql 画布**（`BlockCanvasPage`；`MV_MODEL_SQL_CANVAS_TITLE`）：Model 组与子表分区；子表以 **顶缘标签条**（`role="tablist"`：各子表为 **`role="tab"`** 的可聚焦条，**多于一张时**标签右侧 **×** 删除该子表；**「＋ 子表」** 仍在条末；删除前弹出 **页内 `Teleport` 警告对话框**（**取消 / 确定删除**），**不依赖 `window.confirm`** 以便 Electron / 部分 WebView 下仍能稳定出现）；其下为 **tabpanel** 承载 DDL/DML；当前子表 **DDL · 当前子表列定义**、**DML · 行集（JSON 单元格）** 为两块**可折叠标题栏**（整行按钮展开/折叠，默认展开），内为 DDL/DML 编辑区、**CREATE TABLE** 示意；DML 工具栏含 **WHERE 子串** 筛选与 **「显示只读视图」**（预览**当前子表**行集，与 DML **WHERE 子串**一致筛选）；保存前对**所有子表**补全非空列缺键；可空列清空单元格时省略 JSON 键。
- **mv-view 语义**：**视图基类**，以 `kind` 派生具体类型（含 **全部 `mermaid-*`**、`uml-class` / `uml-sequence` / `uml-activity` / `ui-design` 等，以 `@mvwb/core` 的 `MV_VIEW_KINDS` 与 `MV_VIEW_KIND_METADATA` 为准）。**每个已注册 `kind` 在代码块画布子标签 / 整页代码块画布模式中有对应编辑面**（`BlockCanvasPage`）；中间列**不**展示块卡片预览区。**`mermaid-*`**：允许在 `` ```mv-view `` 后紧随标准 `` ```mermaid`` 镜像段（与 JSON `payload` 同源）；解析为单逻辑块并带 `mermaidMirror` 偏移；**插入** Mermaid 类视图时自动生成两段；**保存** 时 `replaceBlockInnerById` 同步更新镜像正文。
- **mv-model-kv / mv-model-struct / mv-model-codespace / mv-model-interface**：与 `@mvwb/core` 一致；**KV 数据表画布**编辑 `documents[]` 各 JSON 对象；**结构化层次画布**编辑整段 `mv-model-struct` JSON（根组递归）；**代码空间模型画布**（`MV_MODEL_CODESPACE_CANVAS_TITLE`）编辑 `mv-model-codespace` JSON（`workspaceRoot` + `modules[]` 示意）；**接口图模型画布**（`MV_MODEL_INTERFACE_CANVAS_TITLE`）编辑 `mv-model-interface` JSON（`endpoints[]` 示意）。插入对话框 **「设计」**（`mindmap-ui` / `ui-design`）置顶；**「数据库模型（Model）」** 含 `mv-model-sql` / `mv-model-kv` / `mv-model-struct`；**「接口模型（Model）」** 含 `mv-model-interface`；**「软件模型（Model）」** 含 `mv-model-codespace`。
- **围栏代码块「代码块画布」**：Model / View / Map 在 `.md` 中以 **Markdown 围栏代码块**（语言标签 `mv-model-sql` / `mv-model-kv` / `mv-model-struct` / `mv-model-codespace` / `mv-model-interface` / `mv-view` / `mv-map`）存储，块内正文可为 JSON、XML 或纯文本等。**左侧大纲 Dock** 内 **「代码块大纲」** 列出当前文档的围栏块：**每行展示围栏语言与「子类型」**（如 `mv-view` 的 JSON `kind`、`mv-model-sql` 组标题或子表摘要、`mv-model-kv` 标题或文档条数、`mv-model-struct` 标题或根组名、`mv-model-codespace` 标题或模块数摘要、`mv-model-interface` 标题或端点数摘要、`mv-map` 规则条数摘要），以及块 `id`；行可 **选中块**（联动右侧属性），行末 **「代码块」** 在主窗口 **中间列** 文档标签下方以 **子标签** 打开 **`BlockCanvasPage`**（`embedded`），子标签文案含类型+子类型+`id`；即可 **所见即所得 / 结构化** 编辑该代码块内容；同一 `relPath`+`blockId` 重复打开聚焦已有子标签。嵌入画布保存后刷新子标签上的子类型文案。右侧属性 Dock 在已选块时提供与索引等价的 **打开…画布** 主按钮。**保存** 调用 `replaceBlockInnerById` 写回围栏内 JSON/文本，并更新内存 `files`、`sourceEditorText`、Electron 时 `writeWorkspaceFile`。**兼容**：URL `?mvwb_canvas=1&path=&blockId=` 整页画布模式仍保留。主流程不另开 `window.open` / `openBlockCanvas` 窗口。
- **Dock（主工作区）**：**左侧大纲 Dock** 含 **文档章节**、**代码块大纲**、以及选中块后的 **第二段结构概要**（随 `kind` 变化）；**右侧属性 Dock** 含选中块的基本属性、打开画布、编辑 JSON、独立编辑、可折叠完整 JSON。两侧标题栏 **折叠** 与视图菜单 **整侧显示/隐藏** 同前。
- **Markdown 主编辑区**（`section.md-pane`）：**独占中间列「文档」子标签**（与右侧 Dock 并排，**无**「Model / View 块」双栏卡片区）。**三种显示状态**（`mdPaneMode`：`preview` | `rich` | `source`，`h2.md-pane-head` 旁 **预览 / 富文本 / 原始文本** 三钮或右键菜单切换）：**预览** — `MdMarkdownPreview.vue` 调用 **`Vditor.preview`** 只读渲染当前 `files` 中的 Markdown；**富文本** — `MdWysiwygEditor.vue`（Vditor `wysiwyg`）可编辑并与 `files` 同步；**原始文本** — 全宽 `textarea` 绑定 `sourceEditorText`，离开时 **`flushSourceToFiles`** 写回 `files`。默认 **`preview`**。**插入代码块** 仅在富文本或原始文本下可用（预览下禁用）。围栏解析仍驱动 **`blocks`**、左侧代码块大纲与右侧属性，解析警告列表在标题下方。**光标联动属性**（仅富文本 / 原始文本）：偏移与 `ParsedFenceBlock.startOffset`/`endOffset` 对齐；富文本由 `MdWysiwygEditor` 的 **`VditorDOM2Md`** 前缀估算；原始文本用 `selectionStart` + **`selectionchange`**。**插入代码块**（`InsertCodeBlockModal.vue` + `code-block-insert.ts`：左侧 **大纲** 跳转分组；卡片分组——**设计**（`mindmap-ui` / `ui-design`）置顶 / **数据库模型（Model）**（`mv-model-sql` / `mv-model-kv` / `mv-model-struct`）/ **接口模型（Model）**（`mv-model-interface`，卡片名「接口图」）/ **软件模型（Model）**（`mv-model-codespace`）/ **Mermaid UML**（`MV_MERMAID_UML_INSERT_KINDS`：类/序/状态/ER/C4/**architecture** / 需求/ZenUML；**不含** `mermaid-block`——`block-beta` 为通用排版非 UML 标准图类）/ **Mermaid 其他**（`getMermaidNonUmlViewKinds()`：其余全部 `mermaid-*`）/ **PlantUML相关**（含 `uml-class` / `uml-sequence` / `uml-activity` / `uml-diagram`））、`modelRefs` 默认策略、防抖写盘、Vditor CDN、富文本下 `pre.vditor-reset` 样式等同前。
- **浏览器**：`<input webkitdirectory>` 加载 `.md` 到内存 Map；「导出当前」下载单文件。
- **壳检测**：`src/platform.ts` — VS Code 通过扩展注入的 `<meta name="mvwb-shell" content="vscode">`；Electron 通过 `window.electronAPI`。
- **Electron**：`electronAPI.pickWorkspace` / `readWorkspaceFile` / `writeWorkspaceFile` / `openBlockEditor`；块独立窗口通过 `?mvwb_block=1&path=&blockId=` 打开同一 SPA。
