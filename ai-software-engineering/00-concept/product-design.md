# 产品设计：UML Markdown 同步工作台（Vue SDI）

## 愿景

在单一桌面式主窗口（**SDI：单文档界面**，本实现为**单窗口多标签**）中编辑以 Markdown 承载的 UML 与代码契约文件，并通过 `uml.sync.md` 声明目录与同步规则，支持 **UML ↔ 代码** 协同演进。

## 范围（MVP）

- **壳层**：Vue 3 + Vite 的 Web 应用；**SDI** 语义为**一个主工作区窗口**，多文件以**标签页**切换（非 OS 多顶层窗口）。
- **文件类型**：
  - `*.uml.md`：多种 UML 图（以 fenced 代码块为主，默认 **Mermaid**）。
  - `*.class.md`：类定义（结构化 Markdown，供同步与阅读）。
  - `*.code.md`：类以外的函数、宏、全局变量等代码片段；**放在各命名空间根（`namespace_root`）下**，与 `*.class.md` 同树。
  - `uml.sync.md`：命名空间目录、UML 图根目录、代码同步根目录及**同步规则**说明（机器可读块 + 人类可读约定）。
- **编辑**：源码编辑 + UML 侧**实时/按需预览**（Mermaid 渲染）。
- **非目标（MVP）**：全自动双向代码生成器、本地文件系统真实读写（浏览器环境以 **File System Access** 或「粘贴路径」为可选后续）；当前以**工作区演示与格式契约**为主。

## 用户与场景

- 架构师/开发者在仓库中维护 `uml_root`、`namespace_root` 与各 `code_impls` 所指向目录的对齐关系（示例见 `examples/sync-demo/`）。
- 使用 Cursor 规则（`.cursor/rules/uml-code-sync.mdc`）约束 AI 辅助下的同步开发模式。

## UI 壳层（SDI）

- **顶区（页内纵向分区）**：**第一行**为 **标题条**（应用图标 + 产品名）；**第二行**为 **菜单栏**，从左到右 **文件 → 语言（默认至少中/英）→ 主题（跟随系统/浅色/深色分项）→ 窗口（恢复右侧停靠区）→ 帮助**（首项帮助信息+F1，含文件格式与关于）。二者**不得**默认排在同一行；与 `.cursor/rules/window-gui-documentation.mdc`「页内标题条与菜单栏」及「语言与主题菜单」补充一致。
- **客户区**：菜单栏之下为 **横向**：中央 **MDI 工作区**（顶标签 + 下方画布：UML 预览 / 同步表单 / 类与代码占位等）与 **右侧 Dock 列**（**Dock View** 内可并列 **属性**、**文本内容** 两个停靠窗口；**折叠按钮条**贴右缘）。主区与 Dock 列之间有 **竖向分割条**可拖宽；两 Dock 同时展开时，二者之间有 **横向分割条**可调上下比例。具体交互与尺寸以 `02-physical/uml-vue-sdi/spec.md` 为准。
- **状态栏**：窗口最底行；就绪/脏提示、当前路径、点击打开 **Log** 等。

详细几何以 `02-physical/uml-vue-sdi/spec.md` 为准；本仓库不将 `ui.svg` 作为权威线框时，以 **spec + 实现** 对齐为准。
